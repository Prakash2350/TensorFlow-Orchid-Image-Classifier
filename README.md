Project Description

This project uses TensorFlow to build a Convolutional Neural Network (CNN) that classifies images of orchid species found in the northeastern United States and Canada. The main goal of this project was to provide a user-friendly tool for identifying orchid species without the use of dichotomous keys and a familiarity with botanical jargon. The image classifier in the submitted colab notebook is based on the image classification tutorial on the TensorFlow website, and while only 8 classes are presented there, similar code has been used to create models for a variety of species. In all cases, the image classifier outputs probabilities for each species, but can also identify which species is most probable for ease of use. As an arbitrary threshold, confidence scores of 0.95 or greater on distinctive species is considered sufficient. For more challenging species, or photos that are not cropped to focus on flowers, these probabilities were variable. Additional techniques, such as cropping diagnostic features and organizing pretrained models into logic trees, enhance the accuracy for morphologically similar or cryptic species. Neither of these techniques are included in the submission for the sake of brevity.
To aid in collecting data, a URL extractor function was also implemented to quickly generate datasets using iNaturalist.org. Another function (named “identify_orchids”) was used to use pretrained models, thus allowing for the efficient use of multiple models in a single notebook and to avoid retraining. Lastly, confidence score data from the demo model in the final submission was organized in the form of a nested dictionary and used to generate a 3D graph to visualize similarities in appearance.

1. Clone the colab notebook, then copy the orchid_demo_test folder (testing and graph generation) and the orchid_demo folder (for model training and validation). Because the code relies on file paths, please feel free to change these as needed to reflect the location of these copied folders in your Google Drive setup.
2. Once the file paths have been updated to provide access to the necessary folders in Google Drive, run each cell in order from top to bottom. If the necessary dependencies are not installed, the code will fail. A list of these dependencies is included at the end of this section. While the URL_extractor function is not used in the demo notebook, its cell does not need to be run (its purpose was to create the data in the orchid_demo folder). The “identify_orchid” function will be used later, so be sure to run its cell at the start.
3. Mount Google Drive by running its cell and allow access. Be sure to update “dataset_path” to reflect the location of the orchid_demo folder in your Google Drive.
4. After training the model for 10 epochs, change the path in the subsequent cell for “image_path” to direct the model to an image of your choice in the orchid_demo_test folder. In total, the folder contains 17 images (two for each class and an additional cropped photo for one species). Feel free to add photos, but be advised that the model will always attempt to assign photos provided to one of the 8 classes. Also, note that the model does not perform perfectly on the images provided. This is intentional, and helps to show its current strengths and weaknesses (particularly with respect to flower coloration, perspective, zoom, etc).
5. Optionally save your model using “model.save”. Be sure to check the path provided before doing so! Note that the default name for this saved model will be demo_model2. The version made on our end, demo_model1, should be added to your Google Drive for the next part. 
6. Using demo_model1 (or your model, if you choose), adjust “mydemo” accordingly. Use “input_path” to provide a photo of your choice from Google Drive, but preferably from the orchid_demo_test folder. Run the code and enjoy not having to wait for 10 epochs!
7. For part three, it should not be necessary to change “input_path_blank” to proceed. If in doubt, leave it as is.
8. Run the following cells (including importing some more dependencies) to create a library that represents the model’s predictions and confidence scores for all 16 photos (the 17th is not included here), then a graph which helps to visualize their similarities (according to the model). Each node represents a species, and the length of each edge is inversely proportional to its confusion value. Zoom in and move the graph as needed to examine its structure.
9. That’s it! If you would like to test the URL extractor without waiting for the iNaturalist export, you are welcome to use the spiranthes.csv file. Be warned that it is quite large. You will need to import more dependencies, but they are included in the colab notebook just above where the function is called.
